Expand description

A refinement context tracks all the refinement parameters and predicates available in a particular path during type-checking. For example, consider the following program:

ⓘ
#[flux::sig(fn(i32[@a0], i32{v : v > a0})) -> i32]
fn add(x: i32, y: i32) -> i32 {
    x + y
}

At the beginning of the function, the refinement context will be {a0: int, a1: int, a1 > a0}, where a1 is a freshly generated name for the existential variable in the refinement of y.

More specifically, a RefineCtxt represents a path from the root to some internal node in a refinement tree.

pub fn define_vars(&mut self, sort: &Sort) -> Expr

Given a sort that may contain aggregate sorts (tuple or adt), it destructs the sort recursively, generating multiple fresh variables and returning an “eta-expanded” expression of fresh variables. This is in contrast to generating a single fresh variable of aggregate sort.

For example, given the sort (int, (bool, int)) it returns (a0, (a1, a2)) for fresh variables a0: int, a1: bool, and a2: int.
